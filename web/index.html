    GLOSSARY

    Blockchain
    is a continuously growing list of records, called blocks, which are linked and secured using cryptography. Each block typically contains a cryptographic hash of the previous block, a timestamp, and transaction data. By design, a blockchain is resistant to modification of the data. It is "an open, distributed ledger that can record transactions between two parties efficiently and in a verifiable and permanent way.
    Block Hash
    hash of the block where the transaction was in (32-byte).
    Block Number
    block number where the transaction was in.
    Difficulty
    integer of the difficulty for the block.
    Extra Data
    The 'extra data' field of the block.
    From:
    the address the transaction is sent from (20-bytes)
    Nonce (Account)
    the number of transactions made by the sender prior to this one.
    Nonce (PoW)
    hash of the generated PoW (8-bytes) i.e., the random value in a block that was used get the PoW satisfied.
    Gas
    gas provided by the sender.
    Gas Limit
    maximum gas allowed in the block.
    Gas Price
    gas price provided by the sender in Wei.
    Gas Used
    total used gas by all transactions in the block.
    Hash
    hash of the transaction (32-byte).
    Hashing
    means taking an input string of any length and giving out an output of a fixed length. In the context of cryptocurrencies like Ether, the transactions are taken as an input and run through a hashing algorithm (Ethereum uses KECCAK-256 i.e., SHA-256) which gives an output of a fixed length. Hashing is used to index and retrieve items in a database because it is faster to find the item using the shorter hashed key than to find it using the original value. A hash is not ‘encryption’ i.e., it cannot be decrypted back to the original text (it is a ‘one-way’ cryptographic function, and is a fixed size for any size of source text). This makes it suitable when it is appropriate to compare ‘hashed’ versions of texts, as opposed to decrypting the text to obtain the original version.
    Index
    integer of the transactions index position in the block.
    Input
    the data sent along with the transaction.
    Miner
    The address of the beneficiary to whom the mining rewards were given.
    Miners
    For each block of transactions, miners use computers to repeatedly and very quickly guess answers to a puzzle until one of them wins. More specifically, the miners will run the block's unique header metadata (including timestamp and software version) through a hash function (which will return a fixed-length, scrambled string of numbers and letters that looks random), only changing the 'nonce value', which impacts the resulting hash value. If the miner finds a hash that matches the current target, the miner will be awarded ether and broadcast the block across the network for each node to validate and add to their own copy of the ledger. If miner B finds the hash, miner A will stop work on the current block and repeat the process for the next block. Ethereum might not need miners forever, though. Developers plan to ditch Proof-of-Work (PoW), the algorithm that the network currently uses to determine which transactions are valid and protect it from tampering, in favor of Proof-of-Stake (PoS), where the network is secured by the owners of tokens. If and when that algorithm is rolled out, PoS could be a means for achieving distributed consensus that uses fewer resources. It is very difficult for miners to cheat at this game. There is no way to fake this work and come away with the correct puzzle answer. Thaat is why the puzzle-solving method is called 'proof-of-work'.
    Parent Hash
    hash of the parent block (32-bytes)
    Secure Hashing Algorithm
    the SHA was developed by the National Security Agency (NSA) as part of the Digital Signature Program. SHA-256 generates an almost-unique 256-bit (32-byte) signature for a piece of text. SHA3 is the latest member of the SHA family of algortihms.
    SHA3 Uncles
    SHA3 of the uncles data in the block (32-bytes).
    Size
    integer of the size of this block in bytes.
    Smart Contract
    A smart contract is a computer program intended to digitally facilitate, verify, or enforce the negotiation or performance of a contract. Smart contracts allow the performance of credible transactions without third parties. These transactions are trackable and irreversible. The aim of smart contracts is to provide security that is superior to traditional contract law and to reduce other transaction costs associated with contracting.
    State Root
    root of the final state trie of the block (32-bytes).
    Timestamp
    the unix timestamp for when the block was collated.
    To:
    the address the transaction is sent to (20-bytes)
    Total Difficulty
    integer of the total difficulty of the chain until a particular block.
    Transactions
    array of transaction hashes (32-bytes).
    Trustless Service
    Ethereum platform is special in that the smart contracts enable trustless services that obviate the need for trusted third parties in a currency exchange transaction, i.e., disintermediate currency exchange businesses.
    Uncles
    are commonly associated with the Ethereum blockchain and are the equivalent of orphan blocks, but with a slight difference. Uncle blocks are still valid blocks that were mined, but then rejected. However, unlike with an orphan block (Bitcoin) where miners are not rewarded for mining them, miners are in fact rewarded for mining an uncle block.
    Value
    value transferred in Wei.
    Wei
    Ethereum has a metric system of denominations used as units of ether. The smallest denomination (or base unit) of ether is called Wei.
